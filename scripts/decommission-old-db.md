# Удаление старого floraflowerp-db (через 24–48 ч после миграции)

Выполнять **только** когда новая БД уже в бою и всё проверено.

## Чеклист

1. [ ] ERP открывается, логин ок
2. [ ] Заказы / склад / финансы на месте
3. [ ] Витрина knyajestvo работает
4. [ ] `/admin/db-health` на новой БД: логический размер маленький, `wal_keep_size` ≈ 512
5. [ ] Мониторинг `floraflowerp-db2`: диск не уезжает в десятки процентов зря
6. [ ] Прошло **не меньше 24 часов** с переключения

## Удаление

1. Amvera → PostgreSQL → **`floraflowerp-db`** (старый, 25 ГБ)
2. Управление → удалить проект / кластер (подтвердить имя)
3. Удалить его **POSTGRES**-домен, если остался
4. Убедиться, что в переменных ERP/shop больше нет старого хоста
5. Проверить биллинг: платите только за `floraflowerp-db2` (15 ГБ)

## Если что-то пошло не так до удаления

Вернуть `DATABASE_URL` на старый хост в обоих приложениях и написать в поддержку / разобрать логи.
